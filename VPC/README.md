# VPC & Networking

## Summary
Analyzed how AWS services are exposed across network boundaries, focusing on the distinction between internet-facing (public) and VPC-isolated (private) services.

## Key Points
- Public services are accessible over the internet via AWS-managed endpoints
- Private services operate within a VPC and require controlled access (e.g., via routing, NAT, or endpoints)
- VPC acts as a logical isolation layer for workloads

## Outcome
Established a clear understanding of service exposure models, forming the foundation for VPC design and secure architecture.
